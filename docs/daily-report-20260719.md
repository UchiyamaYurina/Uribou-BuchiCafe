# 日報 2026年7月19日

## 本日の作業概要

Uribou & Buchi Cafeの納品版を一旦完成させるため、残り作業時間の整理、制作環境の点検、動作確認で見つかった表示上の問題の修正を行った。

作業計画では、残り51時間のうち24時間程度をFigmaの提案書・ワイヤーフレーム・デザインカンプへ使用し、Webサイトの納品版は24時間を2時間ずつ12コマに分けて進める方針を整理した。進捗サマリー、12コマ計画、30分単位の詳細タスク、納品前チェックリストを含むExcel形式の計画表も作成した。

サイト修正では、軽量化済みのv27を基準に比較版を作り、Reserve、TOP、Access、フッターを順番に確認した。既存の装飾イラストの位置・大きさ・透明度・表示数は変えず、前後関係、文章幅、狭い画面での並び方を中心に調整した。

Reserveページでは、すでに予約ページを表示しているにもかかわらず、同じページのフォームへ移動する固定Reserveボタンが表示されていたため、Reserveページだけから削除した。FAQは既存のグリーンを使い、薄い背景、白い質問カード、緑色の「Q」と開閉記号を追加した。

TOPページでは、背景イラストが文章や営業時間の案内より前に見える問題を調整した。画面を狭くしたときは文章が早めに折り返されるようにし、文字と営業時間の内容を装飾より前へ表示した。

フッターは、固定Reserveボタンとの重なりを避けるため右側だけに大きな余白が設定されており、全体が右寄りに見えていた。左右の余白を同じ幅へ戻し、代わりに固定Reserveボタンがフッターへ近づいたとき、滑らかに上へ移動する仕組みを追加した。

Accessページでは、パソコンとスマートフォンの中間幅で、店舗情報と地図が2列のまま狭くなり、内容が見切れる問題を調整した。画面幅1050px以下では1列へ切り替え、とても狭い画面では項目名と内容も上下に並べるようにした。フッターも同じ幅から中央1列に変更した。

## 学んだこと

- 現在表示しているページへ移動する固定ボタンは、役割が重複する場合がある
- FAQは機能を変えなくても、色、枠、余白、開閉記号で見やすくできる
- `z-index`を大きくしても、親要素の重なり方によって前面にならない場合がある
- 画面幅768px以下だけをスマホ対応にしても、中間幅で窮屈になる場合がある
- 2列が窮屈になる少し前の幅で1列へ切り替えると、見切れを防ぎやすい
- フッターの左右どちらかだけに大きな余白を付けると、全体が片側へ寄って見える
- 固定ボタン側を動かすことで、フッター本体の左右バランスを保てる
- 比較版を番号付きで残すと、変更前と変更後を安全に確認できる
- GitHubの変更コメントには、修正したページと目的を短くまとめると分かりやすい
