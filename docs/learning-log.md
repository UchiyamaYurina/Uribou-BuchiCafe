### 1. 横スクロールと`overflow`

装飾画像を画面の左右から少しはみ出させると、ページ全体の横幅が広がり、横スクロールが発生することがある。

横方向だけを切り、縦方向の装飾画像は表示したい場合は、次のように指定する。

```css
.page-illusts {
  overflow-x: clip;
  overflow-y: visible;
}
```

`overflow: hidden`は横方向だけでなく、縦方向にはみ出した画像も非表示にする。

### 2. `section`の使い方

`section`の中に`section`を入れること自体は、HTMLのルール上は禁止されていない。

ただし、予約・FAQ・来店前のお願いがそれぞれ独立した内容なら、兄弟関係にした方が構造を理解しやすい。

```html
<main>
  <section class="section reserve-section">
    <!-- 予約について -->
  </section>

  <section class="section faq-section">
    <!-- FAQ -->
  </section>

  <section class="section visit-notice-section">
    <!-- ご来店前のお願い -->
  </section>
</main>
```

### 3. FAQに適したHTMLタグ

質問と回答の組み合わせには、説明リストを表すタグが適している。

- `dl`：質問と回答の一覧全体
- `dt`：質問
- `dd`：回答

```html
<dl class="faq-list">
  <div class="faq-item">
    <dt>予約は必要ですか？</dt>
    <dd>事前予約制としています。</dd>
  </div>
</dl>
```

「Q.」「A.」はHTMLへ直接書かず、CSSの疑似要素で表示できる。

```css
.faq-item dt::before {
  content: "Q.";
}

.faq-item dd::before {
  content: "A.";
}
```

デザイン用の文字をCSSで付けることで、HTML本文を読みやすく保てる。

### 4. CSSを書く位置

FAQのCSSは、関連するReserveのCSSの後、FooterのCSSの前へまとめると探しやすい。

```text
Reserve関連
↓
FAQ関連
↓
Footer関連
↓
スマホ用の@media
```

CSSは別の位置でも動作するが、HTMLの表示順に近い順番で整理すると保守しやすい。

### 5. セクションの余白

共通の上下余白は、`.section`の`padding`で指定している。

```css
.section {
  padding: 90px 28px;
}
```

この場合、最初の数値が上下、2番目の数値が左右を表す。

```css
padding: 上下 左右;
```

複数の`section`を連続させると、前の`section`の下余白と、次の`section`の上余白が続くため、間隔が広く見える。

特定のページだけ狭くする場合は、個別に上書きする。

```css
.reserve-section,
.faq-section,
.visit-notice-section {
  padding-top: 60px;
  padding-bottom: 60px;
}
```

ローカルでは直っていても、公開サイトではブラウザに古いCSSが残ることがある。

PC版Chromeでは、最初に次の強制再読み込みを試す。

```text
Ctrl + F5
```

開発中は、Chromeの開発者ツールを使う方法もある。

1. `F12`を押す
2. `Network`を開く
3. `Disable cache`にチェックする
4. 開発者ツールを開いたまま再読み込みする

CSSのURLにバージョンを付ける方法もある。

```html
<link rel="stylesheet" href="css/style.css?v=20260713">
```

この方法は一般的だが、複数のHTMLで番号が違わないように注意する。制作中は強制再読み込みを使い、最終公開時や大きな更新時だけバージョンを変更すると安全である。

### 1. スクロールアニメーション

今回実装した動きは、要素が画面内に入ったことを検知して表示する「スクロールアニメーション」である。

JavaScriptで表示範囲への出入りを検知し、CSSのクラスを切り替える。

```text
画面外：通常のクラスだけが付いている
画面内：is-visibleまたはis-revealedが追加される
```

CSSでは、画面外の初期状態と画面内の表示状態を分けて設定する。

```css
.piggy-card {
  opacity: 0;
  transform: translateX(-100vw) rotate(-8deg);
  transition: opacity 2s ease, transform 2s ease;
}

.piggy-card.is-visible {
  opacity: 1;
  transform: translateX(0) rotate(-1.5deg);
}
```

### 2. フェードとスライドの違い

- フェードイン：透明な状態から徐々に見える動き
- フェードアウト：表示状態から徐々に透明になる動き
- スライドイン：画面外や離れた位置から入ってくる動き
- スライドアウト：元の位置から画面外へ移動する動き

今回、速く感じていたのは透明度の変化よりも、横方向の移動だった。動きを調整するときは、`opacity`と`transform`を分けて考える。

```css
transition:
  opacity 2s ease,
  transform 2.4s cubic-bezier(0.22, 1, 0.36, 1);
```

### 3. 表示順を作る`transition-delay`

写真、名前、プロフィール文章を同時に表示せず、少しずつ遅らせることで視線の流れを作った。

```text
写真
↓
名前
↓
プロフィール文章
```

```css
.piggy-card.is-visible .piggy-name {
  transition-delay: 0.65s;
}

.piggy-card.is-visible .piggy-profile {
  transition-delay: 1.05s;
}
```

`transition-duration`は動きにかかる時間、`transition-delay`は動き始めるまでの待ち時間である。

### 4. 一行ごとのマーカー背景

文章全体を四角い背景で囲むのではなく、文字のある一行ごとに背景を付けた。

```css
.piggy-marker-text {
  display: inline;
  padding: 0.12em 0.35em;
  background: linear-gradient(
    transparent 10%,
    rgba(255, 248, 232, 0.88) 10%,
    rgba(255, 248, 232, 0.88) 90%,
    transparent 90%
  );
  box-decoration-break: clone;
  -webkit-box-decoration-break: clone;
}
```

`box-decoration-break: clone`を使うと、改行された各行へ`padding`と背景が繰り返し適用される。

### 5. 上スクロール時の動き

下スクロール時はスライドイン、上スクロール時はスライドアウトするようにした。

上スクロール時にも遅延を付けることで、急に消えず、写真・名前・文章がゆったり退出する印象になった。

ただし、すべての要素へ大きな動きを付けると主役が分かりにくくなる。Piggiesを最も大きく動かし、ほかのセクションは移動距離や傾きを控えめにすると、サイト全体のメリハリを保ちやすい。

### 6. スマホ表示と動きへの配慮

画面外から大きく移動する要素は、一時的にページ幅を広げる可能性がある。横スクロールが発生していないか、実機でも確認する。

動きを減らす設定を使用している人へ配慮する場合は、次の指定を用意する。

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 2. `IntersectionObserver`と画面外の要素

Piggiesは初期状態で、次のように左右の画面外へ移動していた。

```css
.piggy-card:nth-child(1) {
  transform: translateX(-100vw) rotate(-8deg);
}

.piggy-card:nth-child(2) {
  transform: translateX(100vw) rotate(8deg);
}
```

このカード自体を`IntersectionObserver`で監視すると、カードが画面外にあるため「画面内へ入っていない」と判定され、表示用クラスが付かない場合がある。

そこで、移動していない`.piggies-section`を監視し、セクションが画面内へ入ったら2枚のカードへ`is-visible`を付けるようにした。

```javascript
if (target === piggiesSection && entry.isIntersecting) {
  piggyCards.forEach((card) => {
    card.classList.add("is-visible", "is-floating");
  });
}
```

### 3. 表示状態と漂う状態を分ける

表示状態とアニメーション状態を別のクラスで管理した。

- `is-visible`・`is-shown`：表示済みであることを表す
- `is-floating`：現在、画面内で漂っていることを表す

画面外へ出たときは`is-floating`だけを外し、表示用クラスは残す。

```javascript
target.classList.remove("is-floating");
```

これにより、上スクロールしても情報は消えず、画面外では不要なアニメーションを止められる。

### 2. 必要なときに動画を読み込む


画面確認では、パソコンとスマートフォンだけでなく、その中間幅も確認する必要がある。

### 6. `min-width: 0`でグリッド内の見切れを防ぐ

CSS Gridの子要素は、内容に合わせた最小幅を保とうとする場合がある。長い住所や地図が入っていると、親の横幅より広がることがある。

```css
.access-grid > div,
.map-embed,
.map-embed iframe {
  width: 100%;
  max-width: 100%;
  min-width: 0;
}
```

`min-width: 0`を設定すると、グリッド内の要素が親の幅に合わせて縮みやすくなる。

### 7. フッターが片側へ寄って見えた原因

以前のフッターには、右側の固定Reserveボタンと重ならないよう、右側だけ大きな余白が設定されていた。

```css
.site-footer {
  padding-right: 270px;
}
```

この方法では重なりを避けられるが、フッターの内容全体が左へ押され、左右のバランスが異なって見える。

今回は左右の余白を同じに戻し、固定ボタン側がフッターを避ける方法へ変更した。

```css
.site-footer {
  padding-right: 34px;
  padding-left: 34px;
}
```

狭い画面ではフッターを1列にして、ロゴ、メニュー、説明、著作権表記を中央へ揃えた。

### 8. 固定ボタンをフッターの上へ移動する

固定Reserveボタンは通常、画面右下へ表示している。フッターが画面内へ入ったときは、フッターの見えている高さを調べ、その分だけボタンを上へ移動する。

```text
フッターが画面外
→ ボタンは通常の右下

フッターが画面内
→ ボタンをフッターの上へ移動

上へスクロール
→ ボタンを元の位置へ戻す
```

JavaScriptでは`getBoundingClientRect()`を使うと、要素が画面のどの位置にあるか確認できる。

```javascript
const footerRect = siteFooter.getBoundingClientRect();
const visibleFooterHeight = Math.max(0, window.innerHeight - footerRect.top);
```

CSSの`transition`を使い、`bottom`の値が変わるときに滑らかに動くようにした。

```css
.floating-reserve {
  transition: bottom .42s cubic-bezier(.22, 1, .36, 1);
}
```

端末で「動きを減らす」が設定されている場合は、移動アニメーションを使わないようにする。
